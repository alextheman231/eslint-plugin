## Description of Changes

<!-- user-editable-section-start -->
- Set `jsdoc/require-jsdoc` to error in `internal/jsdoc`
    - The `@alextheman/components` package has recently added JSDoc to all of its exported features and it is the last one in the organisation to add them.
    - As such, we can now strictly require that any new feature in any project that is intended to use JSDoc must have JSDoc comments.
    - This is not expected to affect any other repository as it is a change to one of the internal configs, which should not publicly be used.
